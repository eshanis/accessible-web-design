WCAG Principles for Accessible Websites
WCAG Principle Description Example
Perceivable The information and user interface components must be presented to users in ways they can perceive. This means that users must be able to view and recognize the information being presented to them. The website, applications and devices should be designed in such a way that the content is easily recognizable by sight, sound and touch. The information should be easily transferrable to a format which makes it easier to view/hear like an audio or Braille format. Using color (success criteria 1.4.1) and contrast (success criteria 1.4.4), a control of text size so that it helps visually impaired. Using text alternatives like captioning (success criteria 1.1.1)or sign language (success criteria 1.2.6). Form elements and buttons should have text labels that describe their purpose. Provide captions for web based videos (Success criteria 1.2.2)
Operable The users should be able to find information, navigate through, and interact with web content. The content should be acessible for differently abled people. The web content should be accessible when using a keyboard or mouth stick and should not be dependent on a mouse for navigation. Differently abled people should be able interact and navigate through the content or website The website should be operable using various options like keyboards (success criteria 2.1.1), mouth stick etc. Use character key shortcuts (success criteria 2.1.4). For timed events, allow more time to read and interact with content when using special agents or devices (success criteria 2.2.1), navigation through things like "go to content link" (success criteria 2.4.4), use page titles (success criteria 2.4.2), headings etc. for easiet navigation. Do not use flashing images to avoid seizures (success criteria 2.3.1).
Understandable Users must be able to understand and follow the information as well as figure out the operation of the user interface without any difficulty. The website or application should be clear in its functionality. Differently abled users should be able to comprehend the content and navigate the website or application easily. The web page or app should have predictable and consistent navigation (success criteria 3.2). If technical words or speacial jargon (success criteria 3) are used, definitions should be provided for these.(success criteria 3.1.3). If abbreviations are used provide links to a glossary (success criteria3.1.4). The navigation for the website or app should be intuitive.
Robust Content must be robust enough that users are able to access the content using a wide variety of current and future technologies including assistive technologies. The website and application must be rubust so that various browsers and technilogies can be used by differently abled people to access it. If it is frustrating to access, users will not return to the website or application. The users should be able to use their choice of technology to access the website or application. Content should be well-formed so it can be analyzed successfully by browsers and other user agents such as screenreaders (search criteria 4.1.1). Assistive technology users are made aware of new status messages added to the page (search criteria 4.1.3).
